# Security Policy

## Reporting a Vulnerability

If you discover a security issue in repository automation, scripts, or workflow configuration,
please do not open a public issue first.

Use GitHub Security Advisories to report privately:
<https://github.com/eyupk3/awesome-expo-react-native-dx/security/advisories/new>

If the advisory form is unavailable, open a minimal issue without exploit details and mention
that you need a private security channel.

Include:
- Description of the issue
- Reproduction steps
- Potential impact
- Suggested remediation (if available)

We aim to acknowledge reports within 72 hours.

## Scope

This policy covers:
- CI workflows
- Validation and generation scripts
- Dependency and automation configuration
